Puzzled Golf
Game Design Document

by Siyuan Zou, Kristy Tan, and Billy Luy

INTRODUCTION

This is a game design document for the game “Puzzled Golf”, which is a simple 2D top-down game. This document is constructed to organize each component of the game. The game will deploy basic animation, tiled levels, collision detection, physics engine, and more other 2D gaming techniques.


TECHNOLOGY

The game is going to be developed through Phaser paired with typescript. It is going to be deployed on Firebase and version controlled with git on GitHub. All levels will be made with Tiled.


BACK STORY

After failing to graduate college, you try to persuade your parents to let you pursue pro golf. Because your parents don’t really understand all the logistics behind pro golf, you decide to take up mini golf instead. On your way home from your last failed final, you see a poster advertising a puzzle solving mini golf game with gold stars and a tv show shooting to go along with it. You decide to take up this challenge and impress your parents with your wits and golf skills. See if you can solve these puzzles and win the gold stars!


OBJECTIVE

Clear all the levels with three stars, solving puzzles and trying to sink the golf ball in as little strokes as possible.


GAMEPLAY

The user will try to sink a golf ball into the designated hole of the level, by various puzzles along the way. Score will be calculated based on the amount of stroke required for the user to sink the ball.


CONTROLS

The controls for this game will be both the mouse and keyboard when playing the game:

  • ESC - pressing ESC for the keyboard will bring up the In-Game Menu.
  • Holding Left Click on a golf ball will select the golf ball (highlight), allowing the user to drag back indicating how hard they want to hit the golf ball. Releasing will shoot the ball.
  • Mouse - Moving the mouse around while the golf ball is selected will allow the user to change the direction they want to hit the ball.
  • WASD - Move the viewport around the level
  • Right Click - Right clicking while the golf ball is selected will deselect the golf ball.

GRAPHICAL USER INTERFACE

As far as the GUI is concerned, if we consider

  • Splash Screen - The splash screen will be a simple screen displaying our game logo with text at the bottom of the screen telling the user to press to continue, pressing anywhere on the screen will bring the user to the In-Game Menu.
  • Main Menu - The main menu will be a screen with various levels displayed on it. All levels except the first level will be locked. The main menu will also have a question mark on the top right which the user will be able to click the question mark which will display information about how to play the game.
  • Control Screen - The control screen is a screen that will teach the user how to play that game and what mouse and keyboard inputs are used for the game.

  • In-Game Menu - While the game is in progress, the user will have the option to press a menu button displaying the following:
    • Restart Button - The restart button will allow the user to restart the level they are currently on.
    • Menu Button - The menu button will allow the user to return back to the main menu where they can select which level they want to play.
    • Resume Button - The resume button will bring the user back to the game.
    • Help Button - The help button will display a screen with information on how to play the game.
    • Mute Button - The mute button will be used to mute both SFX and game music.
    • High Score - The high score of the current level will be displayed in the center of the game menu represented by three stars and a number depending on the high score of the current level.

  • In-Game GUI - While playing the game, the following will be displayed:
    • Menu Button
    • Current Score
    • Current Level

ARTWORK

All artwork in the game will be original. The following needs to be created:

  • Golf Ball - The golf ball will be a simple sprite, we will need to make artwork for the golf ball itself as well as animation for when the ball is hit, leaving a trail that disappears.
  • Floor and Wall Tiles - Floor and wall tiles will be used to contain the level the user is currently on and all the interactive tiles in the level.
  • Interactive Tiles - Interactive tiles will be tiles that have an effect on either the golf ball or the level and should each have it’s own animation:
    • Doors
    • Water and Lava
    • Switches
    • Pressure plates
    • Lasers
  • Background Tiles - Tiles in the background will be used to display the current theme of the level, behind the ceiling and walls containing the interactable level.

SOUND EFFECTS

Sounds must be made to coincide with each of the following events:

  • Hitting the golf ball
  • Hitting the wall
  • Landing in a hole
  • Landing in terrain
  • Collision between balls
  • Golf ball disintegrating
  • Lasers
  • Pressure plate
  • Doors
  • Completing the level

MUSIC

  • Main menu music
  • Theme track based off of each level.
